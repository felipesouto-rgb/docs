## O que é isso

Esta área é a **fonte única de verdade** sobre o que a Bloxs estrutura. Comercial, Deals, Compliance, Academy e os Copilotos todos apontam para cá — ninguém redescreve produto em outro lugar.

## Não confunda as cinco camadas

O erro mais comum é tratar tudo como "produto". São camadas distintas:

| Camada | O que responde | Exemplos |
| --- | --- | --- |
| Instrumento | Que título jurídico-financeiro? | CR, CRI, CRA, debênture, nota comercial |
| Veículo | Que estrutura carrega o ativo? | FIDC, FIP, FII, FIAGRO, FIF |
| Regime de oferta | Como a oferta é conduzida? | RCVM 160, 161, 88, private |
| Solução comercial | Que problema do cliente resolve? | funding middle-market, supply-chain |
| Camada tecnológica | Sobre qual trilho roda? | plataforma, white-label |

<Info>
  Uma solução combina camadas: um funding middle-market pode usar um **CR** (instrumento) dentro de um **FIDC** (veículo) sob **RCVM 88** (regime). Nomear cada camada certo é o que deixa o Deal Structuring Copilot recomendar sem errar.
</Info>

## Como escolher — o caminho

<Steps>
  <Step title="É financiável?">
    Todo deal começa pelo [Fundability Test](/produtos/fundability-test). Não passou, não há o que estruturar.
  </Step>
  <Step title="Qual estrutura?">
    Passou, vá para a [Selection Matrix](/produtos/selection-matrix): instrumento, veículo e regime.
  </Step>
  <Step title="Confirme o perímetro">
    A estruturação é atividade regulada. Ver [O Perímetro](/regras/originacao-vs-regulada).
  </Step>
</Steps>
