## Conceito

O **FIDC multiclasse** é um FIDC desenhado como **infraestrutura**, não como fundo único. Cada **classe** é uma **caixa dedicada**, com lastro, risco e retorno-alvo próprios, segregados das demais — **sem contaminação entre verticais**. É a forma de escalar: abrir uma vertical (imobiliário, agro, corporativo, infra, special sits) sem constituir um veh novo a cada tese.

  <Card title="Escala sem reconstruir" icon="trending-up">
    Nova tese = nova classe, não novo fundo do zero.

  <Card title="Uma infraestrutura, várias caixas" icon="container">

  </Card>

  <Card title="Roadmap de verticais" icon="map">
    Abre pela classe de maior densidade de oportunidade; as demais entram por fase.

## Quando usar / não usar

<CardGroup cols={2}>
  <Card title="Usar quando" icon="check">
    Há (ou haverá) múltiplas teses/verticais; quer escalar sem reconstruir; segregação de risco entre estratégias é valiosa.
  </Card>

  <Card title="Não usar quando" icon="x">
    Tese única sem perspectiva de novas classes — um FIDC simples basta e custa menos.
  </Card>
</CardGroup>
