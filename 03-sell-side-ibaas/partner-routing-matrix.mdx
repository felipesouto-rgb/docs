<Note>
  **Status: draft — não recuperável por agente.** Roteia cada ICP para perfil contratual, produto provável, economics e entidade executora. O ICP é uma **lente** — todo parceiro cai em Independent Banker (IB), Business Partner (BP) ou Enterprise IBaaS (ENT).
</Note>

## Como usar

1. Identifique o ICP.
2. Leia a linha: perfil provável, o que tem/falta, produto, economics, entidade, risco, próximo passo.
3. Confirme o perfil contratual com [Conduta por Perfil](/02-regulatory-perimeter/conduta-por-perfil).
4. Valide a estrutura com a [Selection Matrix](/05-products-structures-solutions/selection-matrix).

## Economics de referência (Business Partner)

| Etapa | Faixa |
| --- | --- |
| Set Up | até 20% |
| Success Fee | até 80% |
| Colocação (autorizado) | 80% |
| Colocação (repasse por originação) | 20% |
| Apoio Técnico | até 100% |

<Warning>
  Economics detalhado por contrato é **Restricted-Economics**. Esta tabela é referência de enquadramento interno — não compartilhar faixas específicas com o parceiro fora do processo contratual, nem entre parceiros.
</Warning>

## CRM rules

- ICP → campo de segmentação no `Account`.
- Perfil roteado → `Partner-Program-Enrollment`.
- Deal gerado → objeto `Deal` ligado à conta.

## Next action

- Perfil = ENT → escopo de integração \+ piloto.
- Perfil = BP → onboarding \+ primeiro deal via [Fundability Test](/05-products-structures-solutions/credit-structuring-principles/fundability-test).
- Perfil = IB → onboarding de originação (sem trilha de distribuição).

## Red flags

- Encaixar parceiro num perfil só pela economics, ignorando o perímetro.
- Wealth/assessoria assumindo que pode distribuir por ter investidores.
- Advogado/contador tratados como distribuidores.
